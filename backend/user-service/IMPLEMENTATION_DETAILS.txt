8. CACHING IMPLEMENTATION
================================================================================

Configuration: src/main/java/com/clinicbooking/userservice/config/CacheConfig.java

Redis Configuration:
- @EnableCaching: Activates Spring Cache
- TTL: 30 minutes per entry
- Null values: Not cached
- Key prefix: "user-service:"
- Serialization: Jackson JSON

Cache Names:
- FAMILY_MEMBERS_CACHE = "familyMembers"

Usage Pattern:

On Read (getFamilyMembersByUserId):
- @Cacheable(value = FAMILY_MEMBERS_CACHE, key = "#userId")
- First call: Database query + cache store
- Subsequent calls (within TTL): Cache hit (fast)
- After TTL expires: Database query again

On Write (create, update, delete):
- @CacheEvict(value = FAMILY_MEMBERS_CACHE, key = "#userId")
- Invalidates user's cache entry
- Next query hits database again

Benefit: 10-100x faster for cached queries
