PROTECTED ENDPOINTS (Require Valid JWT)
================================================================================

All endpoints not in whitelist require:
  1. Valid Authorization header: "Authorization: Bearer <jwt>"
  2. Valid JWT token (not expired, valid signature)
  3. User exists in database
  4. User account is active (isActive = true)
  5. Optional: @PreAuthorize role/permission checks

Example Protected Endpoints:
  • GET    /api/users/profile
  • GET    /api/users
  • GET    /api/doctors
  • POST   /api/appointments
  • PUT    /api/users/{id}
  • DELETE /api/users/{id}

================================================================================
AUTHENTICATION FLOW
================================================================================

1. USER REGISTRATION:
   POST /api/auth/register
   └─> No JWT required (whitelisted)
       Returns: User details

2. USER LOGIN:
   POST /api/auth/login
   ├─> No JWT required (whitelisted)
   ├─> Validate email/password
   ├─> Generate JWT and refresh token
   └─> Returns: { accessToken, refreshToken, expiresIn }

3. PROTECTED REQUEST:
   GET /api/users/profile (with JWT)
   ├─> JwtAuthenticationFilter intercepts request
   ├─> Extract JWT from Authorization header
   ├─> Validate token (signature + expiration)
   ├─> Load user from database
   ├─> Check user is active
   ├─> Set authentication in SecurityContext
   └─> Endpoint handler receives authenticated request

4. TOKEN REFRESH:
   POST /api/auth/refresh-token
   ├─> Validate refresh token
   ├─> Generate new access token
   └─> Returns: { accessToken, ... }

5. LOGOUT (Optional):
   POST /api/auth/logout
   └─> Typically just client-side (remove token)
       Can implement token blacklisting if needed

ROLE-BASED ACCESS CONTROL
================================================================================

Supported Roles:
  • PATIENT - Regular patient users
  • DOCTOR  - Medical professionals
  • ADMIN   - System administrators

GrantedAuthority Format: ROLE_{ROLE_NAME}
  • ROLE_PATIENT
  • ROLE_DOCTOR
  • ROLE_ADMIN

Usage with @PreAuthorize:
  @PreAuthorize("hasRole('ADMIN')")
  public void deleteUser() { ... }

  @PreAuthorize("hasAnyRole('DOCTOR', 'ADMIN')")
  public void viewAllAppointments() { ... }

  @PreAuthorize("hasRole('PATIENT')")
  public void bookAppointment() { ... }

================================================================================
ERROR HANDLING
================================================================================

Scenario: Missing JWT Token
  ├─> HTTP Status: 403 Forbidden
  ├─> Cause: No Authorization header or header doesn't start with "Bearer "
  ├─> Filter Action: Continue without authentication
  └─> Endpoint: Return 403 (authentication required)

Scenario: Invalid JWT Token
  ├─> HTTP Status: 403 Forbidden
  ├─> Cause: Token signature verification failed
  ├─> Filter Action: Continue without authentication
  └─> Endpoint: Return 403 (authentication required)

Scenario: Expired JWT Token
  ├─> HTTP Status: 403 Forbidden
  ├─> Cause: Token expiration date has passed
  ├─> Filter Action: Continue without authentication
  └─> Endpoint: Return 403 (authentication required)

Scenario: User Not Found
  ├─> HTTP Status: 403 Forbidden
  ├─> Cause: Email from token doesn't exist in database
  ├─> Filter Action: Continue without authentication
  └─> Endpoint: Return 403 (authentication required)

Scenario: User Account Inactive
  ├─> HTTP Status: 403 Forbidden
  ├─> Cause: User isActive flag = false
  ├─> Filter Action: Continue without authentication
  └─> Endpoint: Return 403 (authentication required)

Scenario: Invalid Credentials (Login)
  ├─> HTTP Status: 401 Unauthorized
  ├─> Cause: Email or password doesn't match
  ├─> Filter Action: N/A (login endpoint whitelisted)
  └─> Endpoint: Return 401 with error message
